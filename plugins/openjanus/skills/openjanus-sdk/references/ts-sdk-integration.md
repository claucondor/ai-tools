# TypeScript/Next.js SDK Integration (v0.3)

How to wire `@openjanus/sdk@^0.3.0` into a Next.js (or React) application.

## Install

```bash
npm install @openjanus/sdk@^0.3.0 @onflow/fcl ethers
```

## FCL wallet connection

```typescript

  "discovery.wallet": "https://fcl-discovery.onflow.org/testnet/authn",

```

## Reading a shielded commitment (server component or API route)

```typescript
// app/api/balance/route.ts
import { ethers } from "ethers";
import { JanusFlow } from "@openjanus/sdk/tokens";
import { isIdentityCommitment } from "@openjanus/sdk/crypto";

export async function GET(req: Request) {
  const { searchParams } = new URL(req.url);
  const userEvmAddress = searchParams.get("address")!;

  // Read-only provider — does not need a signer for view calls
  const provider = new ethers.JsonRpcProvider("https://testnet.evm.nodes.onflow.org");
  const flow = new JanusFlow();
  await flow.connect(provider);

  const commit = await flow.balanceOfCommitment(userEvmAddress);

  return Response.json({
    hasShieldedBalance: !isIdentityCommitment(commit),
    // bigint -> string for JSON serialization
    commitX: commit.x.toString(),
    commitY: commit.y.toString(),
  });
}
```

Note: do not serialize `bigint` values directly — JSON does not support them.
Convert with `.toString()`.

## Generating a proof (client-side, in a Web Worker)

Proof generation takes 10-60 seconds and is CPU-intensive. Run it in a Web
Worker to keep the UI responsive:

```typescript
// workers/prove.worker.ts
import {
  buildShieldedTransferProof,
  generateBlinding,
} from "@openjanus/sdk/crypto";

self.onmessage = async (event) => {
  const {
    oldBalance,
    oldBlinding,
    transferAmount,
    transferBlinding,    // pre-generated by caller so it can be shared with recipient
    newBlinding,         // pre-generated by caller so it can be persisted to local store
    wasmUrl,
    zkeyUrl,
  } = event.data;

  const proofResult = await buildShieldedTransferProof({
    oldBalance:       BigInt(oldBalance),
    oldBlinding:      BigInt(oldBlinding),
    transferAmount:   BigInt(transferAmount),
    transferBlinding: BigInt(transferBlinding),
    newBlinding:      BigInt(newBlinding),
    wasmPath: wasmUrl,
    zkeyPath: zkeyUrl,
  });

  self.postMessage({
    proof:        proofResult.proof.map(String),
    publicInputs: proofResult.publicInputs.map(String),
  });
};
```

```typescript
// components/TransferButton.tsx
const worker = new Worker(new URL("../workers/prove.worker.ts", import.meta.url));

const handleTransfer = async () => {
  setStatus("Generating proof...");
  worker.postMessage({
    oldBalance:       localCommit.amount,
    oldBlinding:      localCommit.blinding,
    transferAmount:   "5000000000000000000",      // 5 FLOW in attoFLOW
    transferBlinding: freshBlindingHex,           // share with recipient OOB
    newBlinding:      freshBlindingHex2,
    wasmUrl: "/circuits/confidential_transfer.wasm",
    zkeyUrl: "/circuits/confidential_transfer_final.zkey",
  });
  worker.onmessage = async (e) => {
    setStatus("Submitting...");
    // Submit via the EVM signer OR via FCL (Cadence router) — examples below
  };
};
```

## Submitting via Cadence (FCL mutation)

```typescript
import * as fcl from "@onflow/fcl";
import { TX_SHIELDED_TRANSFER } from "@openjanus/sdk/tokens";

const txId = await fcl.mutate({
  cadence: TX_SHIELDED_TRANSFER,
  args: (arg, t) => [
    arg(recipientEvmAddress, t.String),
    arg(publicInputs.map(String), t.Array(t.UInt256)),
    arg(proof.map(String), t.Array(t.UInt256)),
  ],
  proposer: fcl.authz,
  payer: fcl.authz,
  authorizations: [fcl.authz],
  limit: 9999,
});

await fcl.tx(txId).onceSealed();
```

## Submitting via EVM (ethers v6 signer)

```typescript
import { JanusFlow } from "@openjanus/sdk/tokens";

const flow = new JanusFlow();
await flow.connectWithSigner(senderSigner);   // an ethers v6 BrowserProvider signer

const tx = await flow.shieldedTransfer({
  to: recipientEvmAddress,
  publicInputs,
  proof,
});
const receipt = await tx.wait();
```

## State persistence — the v0.3 contract

v0.3 has no on-chain decryption key. The app MUST persist on the user's device
every commitment it produces, paired with its cleartext:

```typescript
interface CommitRecord {
  user:      string;     // EVM address that owns the commitment
  amount:    string;     // bigint as string (cleartext residual balance)
  blinding:  string;     // bigint as string (the secret blinding factor)
  commit:    { x: string; y: string };   // for cross-checking against chain
  updatedAt: number;
}
```

Options ordered by security:

- **localStorage** — simple, but visible to other JS on the page. Acceptable
  only for a throwaway demo.
- **IndexedDB with Web Crypto wrapping** — encrypt the blinding with a key derived
  from a wallet-signature challenge before storing. Recommended baseline.
- **Server-side encrypted store** — best for production; tie to user session,
  re-derive the encryption key from a wallet signature on each session resume.

Never expose blinding factors to the server in plaintext. Never log them in
analytics events.

After a `shieldedTransfer`, also forward `(transferAmount, transferBlinding)`
to the recipient via an out-of-band channel (encrypted messaging, push, etc.).
Without that delivery, the recipient cannot recover the transferred amount.
